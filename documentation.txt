UseCase "enter a new order"
Предусловие: продавец получил заказ от заказчика.
Главный поток:
Прецедент начинается, когда Продавец выбирает подходящий пункт для создания
нового заказа
Система выводит форму, в которой Продавец может вводить подробности заказа.
Продавец вводит номер заказа, заказчика и что заказано.
Продавец сохраняет заказ.
Система создает новый заказ и сохраняет его в базе данных.
Послеусловие: Заказ сохранен в базе данных магазина.

UseCase "change existing order"
Предусловие: предложение получено от заказа изменения заказа
Главный поток:
Прецедент начинается, когда Заказчик нажимает кнопку, отвечающую за изменение заказов.
Система выводит список текущих заказов, с возможностью выбора.
Заказчик выбирает необходимый заказ для редактирования.
Система выводит форму с информацией, в которой Заказчик может изменять заказ.
Заказчик вводит необходимые изменения.
Заказчик сохраняет заказ.
Система изменяет заказ и сохраняет его в базе данных.
Послеусловие: Существующий заказ изменен и сохранен в базе данных.

UseCase "update inventory discription"
Предусловие: Запрос Менеджера на обновление инвентарного списка
Главный поток:
Прецедент начинается, когда Менеджер выбирает подходящий пункт для обновления
Система выводит соответствующую форму для ввода данных
Менеджер вводит необходимые данные
Менеджер сохраняет изменения
Система вносит изменения в БД
Послеусловие: Инвентарный список был обновлен

UseCase "print inventory discription"
Предусловие: Запрос Менеджера на печать инвентарного списка
Главный поток:
Прецедент начинается, когда Менеджер выбирает подходящий пункт для печати
Система выводит окно для настройки параметров печати
Менеджер корректирует настройки
Менеджер активирует печать
Система отправляет данные на устройство печати
Послеусловие: Инвентарный список был распечатан

UseCase "place an order"
Предусловие: Поступил новый заказ
Главный поток:
Система выводит информацию о новом заказе
Клерк проверяет наличие товара, при отсутствие расширение в useCase "cancel order"
Клерк заполняет документы и готовит товар для отправки
Клерк обновляет статус заказа
Система сохраняет обновленный статус в БД
Послеусловие: Заказ оформлен

UseCase "cancel order"
Предусловие: При оформлении заказа, товара нет в наличии
Главный поток:
Клерк заполняет документы
Клерк обновляет статус заказа
Система сохраняет обновленный статус в БД
Послеусловие: Заказ отменен
